รายงานระบบ Splunk SOC Monitoring Dashboard

ภาพรวมของโปรเจกต์ (Overview)

รายงานฉบับนี้จัดทำเพื่อแสดงผลงานการออกแบบ Security Operations Center (SOC) Dashboard แบบ Real-time ระบบนี้ถูกออกแบบมาเพื่อรวบรวม Log จากหลายแหล่งข้อมูล (Multi-source) และแสดงผลระดับภัยคุกคามตามความรุนแรง โดยตัว Dashboard มีระบบอัปเดตข้อมูลอัตโนมัติ (Auto-refresh) เพื่อให้การเฝ้าระวังเป็นไปอย่างต่อเนื่อง

![ภาพรวมระบบ Splunk SOC Dashboard](01-Detection-Engineering/screenshots/[01-Detection-Engineering/screenshots/splunk-dashboard-overview(5).png])

ส่วนประกอบหลักของ Dashboard

1. สรุปภาพรวมสถานะความปลอดภัย (Security Posture Summary)

ส่วนบนสุดของหน้าจอจะแสดงจำนวนเหตุการณ์ด้านความปลอดภัยทั้งหมด โดยแบ่งตามระดับความรุนแรง (Severity) โดยใช้ระบบสีเพื่อความรวดเร็วในการสังเกต:

ต่ำ (สีเขียว - Low): เหตุการณ์ทั่วไป หรือข้อมูลพื้นฐานที่ไม่มีความเสี่ยง

ปานกลาง (สีเหลือง - Medium): เหตุการณ์ที่ควรได้รับการตรวจสอบ (เช่น การพยายามเข้าสู่ระบบที่ผิดพลาด)

สูง (สีส้ม - High): กิจกรรมที่มีความสุ่มเสี่ยงสูง (เช่น ตรวจพบรูปแบบ SQL Injection)

วิกฤต (สีแดง - Critical): ภัยคุกคามที่เกิดขึ้นจริง หรือการบุกรุกระบบที่ต้องได้รับการแก้ไขทันที

2. ตารางแจ้งเตือนแบบ Real-time (Alert Panels)

Dashboard ถูกแบ่งหมวดหมู่การตรวจสอบออกเป็น 2 ส่วนหลัก:

Alert (ทั่วไป): แจ้งเตือนเหตุการณ์ผิดปกติของระบบ เช่น การโจมตีแบบ DoS Web และการสุ่มรหัสผ่าน Brute Force Login

Alert (Web App): เจาะจงไปที่การโจมตีระดับ Application เช่น SQL Injection และ Brute Force Attack WebApp โดยจะแสดงข้อมูล Client IP และตำแหน่งไฟล์ที่ถูกโจมตี (เช่น /rattaphon.php)

3. สถิติทราฟฟิกข้อมูล (Traffic Statistics)

ตัวเลขขนาดใหญ่ (เช่น 28,129) แสดงถึงปริมาณเหตุการณ์ (Events) ทั้งหมดที่ถูกประมวลผลในช่วงเวลาที่กำหนด ช่วยให้ผู้ดูแลระบบทราบถึงปริมาณ Load ของข้อมูลในโครงสร้างพื้นฐานที่กำลังตรวจสอบอยู่

ข้อมูลทางเทคนิค (Technical Specifications)

รอบการอัปเดตข้อมูล: ทุกๆ 60 วินาที (Auto-refresh)

ข้อมูลสำคัญที่ติดตาม: เวลาที่เกิดเหตุ (_time), ชื่อกฎการตรวจจับ (rule_name), ระดับความรุนแรง (severity), ไอพีผู้บุกรุก (clientip), และไฟล์ปลายทาง (uri)

แหล่งข้อมูลที่เชื่อมต่อ: Windows Event Logs, Web Server Access Logs และ Sophos EDR

รายละเอียดการติดตั้ง (Implementation Details)

คุณสามารถดูโครงสร้างเบื้องหลัง (XML Source) ของ Dashboard นี้ได้ในโฟลเดอร์ dashboard-configs/ ซึ่งจะช่วยให้สามารถนำโครงสร้างนี้ไปติดตั้ง (Replicate) บนระบบ Splunk อื่นๆ ได้ทันที
